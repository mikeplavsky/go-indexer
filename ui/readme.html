CUI Application Frame

The CUI Application Frame is a responsive control that provides your application with the primary branding elements of the masthead and the navigation. The ability to configure some key features, such as the application name, the logo, navigation, and secondary controls/navigation in the header is baked in. This example page demonstrates some of the most common features in any Dell application.

Learn more about the CUI Application Frame.

cui-application-frame element

Configuration and Layout of this page

Navigation

In the AppController controller, you can see how the navigation on this page is configured by creating an array of objects in the controller and then adding that as a property on the $scope. Then we set the navigation-items attribute of the cui-application-frame element to that $scope property.

The CUI Application Frame API Doc outlines the various attributes that you can use to configure the navigation and other features.

CUI About Box

There is an additional controller that demonstrates how to implement the CUI About Box, which is added via the cui-controls element of the CUI Application Frame as a CUI Button control. The button makes use of an internal Angular directive ng-click to trigger the opening of the CUI About Box. This particular CUI control injects a service into its controller when instantiated by passing cuiAboutBox as an argument along with $scope.

You'll notice that the About Box is using a nested AboutBoxCtrl controller inside of the AppController. Using multiple nested controllers in an Angular application is considered a best practice to separate concerns and maintain the single responsibility principle. The CUI About Box in this example is very basic. You'll notice that there are provided templates for the basic tabs.

CUI CSS

Throughout this page you'll notice various class names. Most notable here is the use of the CUI Layout Grid. It is a basic CSS layout grid that should provide your application with solid structure. Please consider that if you are ending up needing to nest grid-unit upon grid-unit, there is most likely a better alternative to your layout.
